# Development Logger Skill

이 스킬은 개발 세션 중 주요 변경 사항, 의사결정, 남은 과제를 기록하여 `docs/daily_logs/` 디렉토리에 저장하는 역할을 담당합니다.

## 스킬 사용 시점
- 사용자가 "개발 일지 써줘", "오늘 한 거 정리해줘"라고 요청할 때
- 중요한 기능을 구현한 후 세션을 마무리할 때
- 복잡한 디버깅이나 리팩토링 후 맥락 보존이 필요할 때

## 저장 규칙
1.  **저장 경로**: `docs/daily_logs/`
2.  **파일 명명 규칙**: `YYYY-MM-DD_{Title}.md`
    - 예: `2026-01-16_setup-agent-skills.md`
    - 날짜는 사용자 로캘 기준 오늘 날짜를 사용합니다.
    - Title은 내용을 대표하는 영문 소문자(kebab-case)를 권장합니다.

## 로그 구조 (Template)

작성 시 아래 포맷을 엄격히 준수하십시오. 특히 **'Context for AI'** 섹션은 미래의 세션에서 프로젝트를 이어받을 때 핵심적인 역할을 합니다.

```markdown
# [YYYY-MM-DD] {Title}

## 1. 📝 요약 (Summary)
오늘 수행한 작업의 핵심 목표와 결과를 1-2문장으로 요약합니다.

## 2. 🚀 주요 변경 사항 (Key Changes)
작업한 파일과 구체적인 변경 내용을 나열합니다.
- **Feat/Fix/Refactor**: 작업 내용 설명
  - `path/to/modified_file.py`: (변경의 핵심 내용)

## 3. 🧠 기술적 맥락 (Context for AI)
*이 섹션은 다음 세션의 AI를 위해 작성됩니다.*
- **구현 의도**: 왜 이런 방식으로 구현했는지 설명 (Design Rationale)
- **트레이드오프**: 선택하지 않은 대안이나 감수한 단점
- **중요 변수/설정**: 기억해야 할 환경 변수나 설정 값

## 4. ✅ 남은 과제 (Next Steps)
- [ ] 구체적인 다음 작업 아이템 1
- [ ] 구체적인 다음 작업 아이템 2
```

## 작성 절차
1.  오늘의 작업을 분석하여 **파일 이름**과 **핵심 내용**을 먼저 사용자에게 제안합니다.
2.  사용자가 승인하면 파일을 생성합니다.
