# GitHub Expert Skill

이 스킬은 프로젝트의 얼굴인 `README.md`를 최적화하여 기술적 가치를 증명하고, 엄격한 **Git 워크플로우**를 통해 작업 내용을 안전하게 백업 및 관리합니다.

## 스킬 사용 시점
- **백업 및 저장**: 작업 중간이나 종료 시 안전하게 코드를 저장(Commit/Push)해야 할 때
- **문서화**: `README.md` 작성, 리뉴얼 또는 프로젝트 구조를 설명할 때
- **포트폴리오화**: 외부 공개를 위해 리포지토리를 매력적으로 다듬어야 할 때

## 1. 안전 및 버전 관리 (Backup & Version Control)
**"모든 작업은 기록되어야 하고, 언제든 복구 가능해야 한다."**

### 1.1 상시 백업 규칙 (Always Backup)
- **선행 커밋 (Pre-work Save)**: 대규모 리팩토링이나 위험한 변경 전에는 반드시 현재 상태를 커밋합니다.
- **작업 후 커밋 (Post-work Save)**: 하나의 기능 구현이 완료되거나, 세션이 종료될 때는 반드시 커밋을 제안합니다.
- **원격 저장소 동기화**: 로컬 커밋 후에는 가능한 한 `git push`를 통해 GitHub 원격 저장소에 백업하여 로컬 데이터 유실에 대비합니다.

### 1.2 의미론적 커밋 메시지 (Semantic Commits)
히스토리 파악을 위해 다음 컨벤션을 따릅니다.
- `feat: `: 새로운 기능 추가
- `fix: `: 버그 수정
- `docs: `: 문서 수정 (README 등)
- `style: `: 코드 포맷팅, 세미콜론 누락 등 (로직 변경 없음)
- `refactor: `: 코드 리팩토링 (기능 변경 없음)
- `test: `: 테스트 코드 추가/수정
- `chore: `: 빌드 태스크 업데이트, 패키지 매니저 설정 등

### 1.3 원자적 커밋 (Atomic Commits)
- 여러 가지 변경 사항을 한 번에 뭉뚱그려 커밋하지 마십시오.
- **하나의 커밋은 하나의 논리적 변경**만 담아야 합니다. (예: 버그 수정과 새 기능 추가는 분리)

## 2. 리포지토리 최적화 가이드 (Scout Appeal Strategy)

### 2.1 시각적 임팩트 (The Hook)
방문자가 처음 3초 안에 프로젝트를 이해하고 흥미를 느껴야 합니다.
- **Hero Image**: 프로젝트를 대표하는 로고나 세련된 배너 이미지를 최상단에 배치하십시오.
- **Badges**: 기술 스택, 라이선스, 빌드 상태, 버전 등을 뱃지(Shields.io)로 시각화하여 전문성을 보여주십시오.
- **Demo GIF/Video**: 텍스트 설명보다 실제 동작하는 화면(GIF) 하나가 더 강력합니다.

### 2.2 README 구조화 (Structure)
다음 순서를 기본으로 하되, 프로젝트 성격에 맞춰 조정하십시오.
1.  **Project Title & One-liner**: 프로젝트 이름과 핵심 가치를 한 문장으로 요약.
2.  **Key Features**: 핵심 기능을 3~5개로 요약 (이모지 활용).
3.  **Tech Stack**: 사용된 기술을 프론트엔드/백엔드/인프라로 나누어 명시 (아이콘 활용 권장).
4.  **Architecture**: 시스템 아키텍처 다이어그램(Mermaid 등)을 포함하여 구조적 사고력을 증명하십시오.
5.  **Getting Started**: 누구나 따라 할 수 있는 설치 및 실행 가이드 (Prerequisites -> Installation -> Usage).
6.  **Screen Shots**: 주요 기능별 스크린샷.

### 2.3 전문성 강조 (Engineering Depth)
단순한 코드 저장소가 아님을 보여주십시오.
- **Troubleshooting**: 겪었던 문제와 해결 과정을 담은 블로그 링크나 위키 페이지를 연결하십시오.
- **Project Structure**: 주요 디렉토리 구조를 트리 형태로 보여주고 역할을 설명하십시오.

### 2.4 관리 및 기여 (Hygiene)
- `.github/` 폴더를 활용하여 Issue/PR 템플릿을 제공하십시오.
- 명확한 라이선스 표기와 기여 가이드(CONTRIBUTING.md)를 작성하십시오.
